# Browser Invocations Carry a Debug Toggle

Browser Workbench keeps a sticky Debug checkbox in the Side Panel footer, alongside output language and text size. The preference is stored per browser profile in `chrome.storage.local` and resolved fresh at each toolbar or Page Conversation Send/Retry gesture, mirroring how output language is resolved (ADR 0043). When on, protocol v3 carries `debug: true` on the request; the Native Host forwards it straight into the existing `RunKernelSessionInput.debug` seam already shared by the CLI, so a Page Brief or Page Answer Session writes the same Debug Transcript (ADR 0018) to `.forgelet/debug/<sessionId>.jsonl` under the resolved Workspace Profile. Toggling Debug never reruns or rewrites a completed turn; it affects only the next launch. There is no in-Side-Panel Debug Transcript viewer: `forge debug show <sessionId>` remains the only reader, unchanged.

We rejected building a dedicated in-panel viewer because Debug Transcripts already have one, and adding a second reader would fork the presentation of the same content; we rejected making Debug part of the persisted Page Conversation Projection because, like output language, it is a launch-time preference snapshotted per gesture, not conversation state.
