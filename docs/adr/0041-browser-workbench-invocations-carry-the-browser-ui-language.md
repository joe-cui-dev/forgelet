# Browser Workbench Invocations Carry the Browser UI Language

The Browser Workbench builds the Learning task itself, so without a language signal every Learning Pack comes out in English even for users whose browser runs in another language. The extension will therefore read the browser UI language (`chrome.i18n.getUILanguage()`, falling back to `navigator.language`) at invocation time and send it as an optional top-level `uiLanguage` field in the strictly validated invocation payload; the Browser Workbench appends a language directive to the Learning task so body text follows that language while the five Learning Pack headings stay English for the normalizer. The field is a narrowly validated BCP 47-shaped tag on both sides: the extension drops anything the native host would reject, and the native host rejects anything else rather than passing free text toward the prompt. The browser UI language was chosen over a per-Workspace-Profile setting because it is zero-configuration and follows the user automatically; the optional field leaves room for a profile-level override later without a protocol version bump. The `forge learn` CLI path is unchanged — there the user writes the task and its language directly.
