# Rolling Summary Narrative Is Bounded and Clipped, Not Retried

The fold prompt asked for concision but nothing enforced it, and each fold feeds the previous narrative back in, so the narrative grows monotonically across folds. The narrative now gets a byte cap of 25% of `activeContext.maxConversationBytes`, mirroring the Fact Ledger's 25% (ADR 0023). The symmetry is an invariant, not a coincidence: the whole Rolling Summary stays at or under 50% of budget — exactly the fold low-water ratio — so the Rolling Summary alone can never exhaust the fold target, and ADR 0019's stop condition can only fire when the protected turns are genuinely too large.

On overflow the narrative is deterministically clipped at the cap with an appended truncation note and a Trace event. Treating overflow as a fold failure (feeding ADR 0022's degraded-fold counter) and retry-with-firmer-instruction were both rejected: they discard a usable summary and spend an extra model call to avoid tail loss that the architecture already priced in — ADR 0019 assigns the narrative only task continuity, while deterministic facts live in the Ledger and the Trace. Overflow also signals model verbosity, not model unavailability, and does not belong in the same failure counter.
