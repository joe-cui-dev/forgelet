# Writing Project Membership Lives in a Manifest With a Deterministic Continuation Head

Writing Artifacts sit flat in `.forgelet/writing/` and the catalog is derived from Traces plus a directory scan, so the obvious way to group artifacts into a long-form work would be more trace derivation: record a project slug at session start and group at read time. That was rejected because the catalog already has to tolerate missing or corrupted Traces (its `untracked` and `missing` states exist for exactly that), and a Writing Project is a long-lived structure a user builds over months — its membership should not dissolve when a Trace file does. Instead, each Writing Project has a Project Manifest at `.forgelet/writing/projects/<slug>.json` that is the sole source of truth for membership: an ordered member list plus an explicit Continuation Head. Traces still record the project slug and a `writing_project_updated` event when the manifest changes, but as evidence, never as authority.

The head exists because "continue from the newest artifact" is wrong for long-form work: revising chapter 3 after drafting chapter 10 produces the newest file, and both timestamp order and list-append order would silently redirect "write chapter 11" to continue from the chapter 3 revision. The head advance rule is deterministic and never passes through a model: new artifacts always append to the member list, but the head moves only when the Session's continuation source was the current head. Continuing the head advances it; revising an earlier member never does. Users can edit the head directly — the manifest is theirs.

Membership changes are explicit, matching ADR 0017's posture: projects are created only by `forge write projects create <slug>` (an unknown `--project` slug is an error listing existing projects, so a typo cannot silently fork a work), a `--continue` target that is not a member is an error rather than an implicit enrollment, and the only automatic membership change is that a project Session's own output artifact appends to the list. Manifest loading validates deterministically (member paths under `.forgelet/writing/`, head ∈ members) and reports the violated constraint instead of degrading silently.
