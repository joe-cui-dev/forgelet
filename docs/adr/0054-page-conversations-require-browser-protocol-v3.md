# Page Conversations Require Browser Protocol v3

Browser protocol v3 introduces explicit Page Conversation identities, root/follow-up/Retry request variants, and Page Answer terminal frames, and the Native Host rejects all earlier protocol versions instead of preserving v2 root-only compatibility. Extension/host mismatch must fail with a typed protocol error and recovery guidance to rebuild Forgelet, reload the unpacked extension, and rerun `forge browser install-host` when the installed host path or extension identity requires it. Side Panel projection state is likewise marked `schemaVersion: 3`; older or unversioned state is discarded with a prompt to start a new toolbar capture, while existing Session Traces and capture files remain untouched. This deliberate hard cut prevents optional-field negotiation or synthetic migration from producing a summary that appears successful while silently lacking continuation semantics.
