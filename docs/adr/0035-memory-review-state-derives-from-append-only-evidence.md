# Memory Review State Derives from Append-Only Evidence

Project Memory Review never stores current state: every status it reports is derived at read time from append-only records. Suggestions and decisions live in two append-only JSONL files; the Memory Decision Log is the sole authority on decisions, first record per suggestion wins, and the log append is the commit point of an acceptance. Because the decision record precedes the Durable Memory write, it records only the intended write — the Rendered Memory Block's hash and destination — and a second record kind, the Memory Write Record, confirms what was actually written; a decided-but-unwritten crash gap (a Memory Write Gap) is therefore derivable from the log alone and closes on idempotent re-accept. Preview-to-write integrity is structural, not checked: show and accept share one deterministic renderer whose only input is the immutable suggestion record, so previewed bytes and written bytes cannot diverge within a code version. The user-editable Durable Memory file and the source Trace are never authorities — Trace Corroboration is informational and never gates a decision, and the memory file serves only as an anti-duplication guard during repair. Alternatives rejected: mutating a status field in place (whole-file rewrites of evidence), scanning the memory file to detect gaps (legitimate user edits per ADR 0007 would read as corruption forever), and writing memory before the log (a crash would leave an effect no evidence explains — evidence must commit before the effect it accounts for).
