# Unattended Sessions Are Pre-Authorized by an Effect Envelope

Background Sessions cannot rely on per-action confirmation because the confirmer is away, and a background Session that stops at its first durable write has almost no value for write-heavy work like coding. Instead of relaxing risk tiers globally or queueing unapplied proposals that break verify-after-edit loops, the user declares an Effect Envelope at Session start: the write scope, command allowlist, and budget within which confirmable actions are auto-approved and recorded in Trace. Actions outside the envelope still require an explicit user decision, and destructive or secret-touching actions stay denied regardless of the envelope. This moves the ADR 0011 confirmation moment to Session start and bounds it, rather than removing it.
