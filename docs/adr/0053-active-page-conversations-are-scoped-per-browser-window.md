# Active Page Conversations Are Scoped per Browser Window

Browser Workbench keeps one active Page Conversation projection and reattachment pointer per Chrome window. A toolbar gesture captures the active tab and replaces that window's active projection with a new conversation, while other windows remain unchanged; later tab navigation or closing the captured tab does not end the conversation. Each window permits one in-flight attempt, but different windows may run concurrently, with Native Host frames and cancellation routed by `invocationId`. If the same window receives another toolbar gesture while its attempt is running, Browser Workbench re-focuses that attempt and asks the user to wait or Stop; it does not capture, launch, implicitly cancel, or hide work. The extension stores the window mapping in `chrome.storage.session` rather than treating the browser profile or source tab as the owner.
