# Trace Events Are a Closed Vocabulary with Additive Evolution

The Trace is Forgelet's audit evidence backbone — Sessions, Effect Envelopes, memory provenance, the Decision Queue, and Session Continuation are all defined in terms of Trace evidence — yet `TraceEvent` was `{ type: string; payload: Record<string, unknown> }`. Roughly forty event kinds existed only implicitly: producers assembled payloads free-form at each `appendTrace` call site, eighteen consumer modules re-derived payload shapes by duck-typing (silently degrading to empty values on drift), and tests pinned raw JSONL sequences because no better surface existed. Every rung of the autonomy ladder adds Trace consumers, so the cost of the implicit schema compounds.

We decided the Trace Vocabulary is closed: one discriminated union, declared in the trace module, covers every event kind a Session may record — kernel and workflow events alike. Producers write through a mapped-type `appendTrace` signature, so each call site's payload is checked against the vocabulary without changing call shape. Readers validate the event envelope (`type`, `ts`, `sessionId`, `payload`) and vocabulary membership; a line that fails either check becomes an explicit unknown-event value that is preserved for display and never silently dropped — and never masquerades as evidence. Payloads of recognized kinds are typed by the vocabulary rather than re-validated field-by-field: the writer is this same program, so the real reading-time threats are version skew and truncation, which the envelope check and the unknown-event value address.

The vocabulary evolves additively instead of carrying version numbers: payload fields are never renamed or repurposed, new fields must be optional, and a change in meaning mints a new event kind while the old one remains declared as history. Both directions of version skew then route to existing mechanisms — an event kind from a newer Forgelet falls into the unknown-event value for old readers, and an old Trace merely lacks optional fields for new readers. For an audit format this is a feature: two eras of a meaning are permanently distinguishable by name.

We rejected an extensible per-workflow vocabulary (generic machinery serving three slow-growing workflows, at the cost of the enumerable-evidence property: review surfaces should be able to state exactly what may appear in a Trace), runtime schema validation via a library (the project's first runtime dependency, and a second source of truth that can drift from the vocabulary), and per-event or per-file version fields (a version number without a migration machine is a field nobody checks). The on-disk JSONL format is unchanged and no migration runs; a consequence of the typed vocabulary is that the session lifecycle fold — status, task, route, timestamps, final summary — consolidates into the sessions module as the single authority, where four consumers previously each re-implemented it.
