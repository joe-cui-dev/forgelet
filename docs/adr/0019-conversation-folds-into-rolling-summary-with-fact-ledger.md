# Conversation Folds Into a Rolling Summary With a Fact Ledger

ADR 0013's deterministic Observation Digests only govern tool observations, so long dogfood Sessions still outgrow the context window through assistant messages and residual overage that compaction records but never resolves. The compaction budget therefore widens from observation bytes to the whole conversation array (assistant and tool messages; Context Attachments stay excluded because system-mutated attachments would silently change user-approved material — that is an admission concern, not a compaction concern). The config key becomes `activeContext.maxConversationBytes`; it must not be named after Active Context itself, because Active Context includes attachments and this budget deliberately does not.

When digests cannot hold the conversation under budget, the oldest whole turns fold into a single Rolling Summary at the head of the conversation. This revisits ADR 0013's exclusion of model-generated semantic summaries, but keeps its core intact: the Rolling Summary pairs a model-generated narrative with a machine-assembled Fact Ledger built deterministically from the folded Observation Digests, so deterministic facts (paths, hashes, ranges, exit codes, changed files) still never pass through a model. Folding is whole-turn only because provider APIs require tool results to stay paired with their assistant tool calls. A rolling single block was chosen over a checkpoint chain because chains are unbounded and would exempt themselves from the very budget they serve; repeated re-encoding loss is acceptable because deterministic facts live in the Ledger and the Trace, leaving the narrative to carry only task continuity.

Folding runs on hysteresis: it triggers above the budget high-water mark and folds down to a low-water mark (50% of budget), never touching the most recent protected turns (`activeContext.protectedRecentTurns`, default 3), so a fold buys several quiet turns instead of paying a summarization call every turn. The summarization call uses the Session's current route model, counts against token and cost budgets but not model turns (turns measure task progress, and counting overhead would make long sessions die faster), and holds no budget privilege — if the fold itself breaches a budget line, the Session stops. A fold is atomic: the conversation is only replaced after a non-empty summary is returned; on failure the Session carries the overage and retries next turn. The full Rolling Summary text is recorded in the fold Trace event — it shapes all subsequent model behavior, so it is review evidence, not raw content storage. Deterministic dropping of old turns and tool-side re-read blocking were both rejected: silent drops conflict with the audit posture, and after a fold, re-reading a file is often legitimate. If the protected turns plus the Rolling Summary alone exceed budget, the Session stops rather than shrinking the protected region.
