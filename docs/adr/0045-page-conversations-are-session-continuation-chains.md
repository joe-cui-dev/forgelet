# Page Conversations Are Session Continuation Chains

A Page Conversation is presented as one continuous Side Panel exchange, but its Page Brief and each follow-up answer are separate Sessions in one linear Session Continuation chain. Every submitted follow-up creates a child of the current conversation head instead of reopening the completed Session or starting an unrelated run. This preserves immutable per-Session Traces and the existing answer-once browser execution boundary while giving the browser a conversational projection over explicit lineage.
