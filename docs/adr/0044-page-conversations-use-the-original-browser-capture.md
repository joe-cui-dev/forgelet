# Page Conversations Use the Original Browser Capture

A Browser Workbench Page Conversation remains bound to the immutable page capture created by its toolbar gesture. Follow-up turns may use that capture and the conversation so far, but they do not silently recapture a changed page, read workspace content, or query the public Web. The Side Panel keeps the capture's title, URL, capture time, and partial-content status visible even after the user navigates away, and the conversation remains usable. Capture serialization records whether per-block or total byte limits truncated the source, and that coverage metadata follows the Context Attachment and Trace. Before every follow-up, the Native Host reloads the persisted capture inside the pinned workspace and verifies its capture identity and content hash; missing content or an integrity mismatch rejects the launch before a child Session starts, with no fallback to changed file bytes or the disposable browser projection. If the root Page Brief attempt fails, Retry launches a new root attempt against the same persisted capture; only another toolbar gesture captures the current tab into a new Page Conversation. The Side Panel switches to the new conversation while old Traces remain intact. This keeps answers source-auditable and avoids quietly mixing page versions or authority boundaries in one conversation.
