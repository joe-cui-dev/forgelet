# Page Answers May Draw on Model Background Knowledge

A Page Answer's `Answer` text is no longer confined to restating the captured page: it may draw on the model's own background knowledge to inform, explain, or contextualize the response, with depth matched to the question — thorough for open-ended questions, concise for narrow factual ones. This does not touch ADR 0044's I/O boundary: follow-up turns still never silently recapture the page, read the workspace, or query the public Web. The Answer body carries no source-attribution markers between page-derived and background-knowledge content; that judgment is left to the reader, not machine-verified. The Evidence contract from ADR 0048/0049 is unchanged in structure and machine verification — up to three excerpts mechanically matched against the captured page, or the not-found sentinel — but its meaning narrows: Evidence now verifies only the parts of the Answer grounded in the captured page, and the not-found sentinel means no passage in the page backs the Answer (for example because it relies on background knowledge), not that the question went unanswered. The Answer must not use Markdown headings, so a longer Answer stays inside the two-heading Page Answer shape without colliding with the strict section parser.
