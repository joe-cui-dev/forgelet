# Page Conversations Run One Child at a Time

A Page Conversation permits one in-flight Session attempt only: its input becomes available after a root Page Brief completes, Send is disabled while a child runs, and Stop cancels only the active attempt. The Page Conversation Head advances only when an attempt completes successfully, so stopped and failed runs remain auditable without contaminating later history. The Side Panel retains each failed or stopped question as a status card with its terminal reason, Session identity, Retry action, and any text already shown while streaming. That text is labelled as an unverified streamed draft, remains bounded by projection eviction, and is excluded from Page Conversation History and all continuation input. A child Retry creates a new child from the unchanged head with the same question; a root Retry creates a new root Session against the same capture while the conversation has no head. Retry never reopens, overwrites, or appends to a failed Trace. Forgelet will not queue unseen dependent questions or create parallel branches from the Side Panel in this slice.
