# Page Conversation History Reuses Active Context Compaction

Each Page Conversation child Session reconstructs its ordered history from ancestor Traces, keeps recent turns exact, and lets the existing Active Context compaction machinery fold older turns when the Session budget requires it. The immutable browser capture remains a separate Context Attachment, and complete normalized turns remain recoverable from their original Traces. If any ancestor Trace is missing, unreadable, or malformed, the launch fails before creating a child Session with `conversation_history_unavailable`; Page Conversation does not use generic degraded continuation or reconstruct authority from the browser projection. Forgelet will not add a second durable conversation-summary store, silently discard oldest semantic turns, or stop an otherwise valid follow-up solely because the uncompressed lineage exceeds one model context.
