# Browser-Launched Sessions Use Approved Workspace Profiles

Browser-launched Sessions cannot inherit a terminal working directory and must not accept arbitrary filesystem paths from the extension. Forgelet will resolve a `workspaceProfileId` against user-approved local Workspace Profiles; the Side Panel may show and remember one default, but recently used workspaces do not become approved automatically, and a missing or removed profile rejects the launch before a Session starts. Only local `forge browser profiles` CLI commands may approve the current working directory, list profiles, select a default, or revoke a profile; the extension can only list and select opaque approved identities, and Native Host installation never grants workspace authority implicitly. A Page Conversation pins the root Session's profile identity and re-resolves it before every follow-up child launch: changing the browser default never moves the lineage, while revoking or deleting the pinned profile blocks later children. Revocation does not rewrite authority already recorded for a running Session. This preserves explicit workspace authority without binding the extension permanently to one installation-time workspace.
