# Side Panel Reattaches from a Disposable Conversation Projection

During the current Chrome session, Browser Workbench stores a bounded Page Conversation Projection in `chrome.storage.session`: capture identity metadata, root and head Session identities, normalized successful outcomes, failed or stopped attempt cards, and current attempt status. Closing the panel or restarting the extension service worker reattaches from this projection without contacting a new model. If the projection exceeds its deterministic byte budget, it keeps the source header, Page Brief, Page Conversation Head, and current attempt, then evicts the oldest successful turns and terminal attempt cards while showing that earlier turns remain in Session Traces. The projection is disposable presentation state, never the sole input to a continuation; workspace capture files and immutable Session Traces remain authoritative for source content, complete normalized history, and lineage.
