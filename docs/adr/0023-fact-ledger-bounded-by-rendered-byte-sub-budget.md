# Fact Ledger Is Bounded by a Rendered-Byte Sub-Budget

The Fact Ledger appended commands without deduplication and accumulated file ranges forever, so on long Sessions the Ledger itself consumed the conversation budget and could spuriously trigger ADR 0019's stop condition ("protected turns plus Rolling Summary alone exceed budget") — a signal meant to say the *active work* no longer fits, not that the ledger kept a flat transaction log. Bounding the Ledger loses no audit evidence: the Ledger is Active Context, which may be compacted without changing the Trace, and every evicted fact remains in the Trace.

The bound is deterministic, in three layers, honoring the Ledger's never-passes-through-a-model constraint. First, lossless structural deduplication: commands dedupe by command string keeping the last exit code plus a run count, and file ranges merge into spans. Second, a rendered-byte sub-budget of 25% of `activeContext.maxConversationBytes`; when exceeded, entries evict in fixed order — oldest commands first (lowest freshness value, recoverable from Trace), then oldest file-read entries (re-reading after a fold is legitimate per ADR 0019), while changed files are never evicted (they keep the model from clobbering its own work and are naturally bounded). Third, any eviction appends a deterministic truncation note to the rendered Ledger, so like a Degraded Fold (ADR 0022) it is never a silent drop. A pure count cap (keep last N entries) was rejected because the system's budget language is bytes and count caps cannot prevent byte blowups from long commands or paths.
