# Paused Sessions Persist a Pause Snapshot

Pausing in place per ADR 0027 must survive process exit: a background Session pauses when the user is away, and holding a live process hostage until they return loses hours of unattended work to a crash, reboot, or closed terminal. Resuming the same Session requires its model-facing conversation, which nothing durable holds today — Trace records events, not the model exchange (ADR 0012), the Rolling Summary is lossy by design, and the Debug Transcript is an opt-in diagnostic record (ADR 0018). So a pausing Session writes a Pause Snapshot — conversation, Effect Envelope, remaining budget, workflow position — and exits; deciding from the Decision Queue resumes a fresh process from the snapshot. This does persist model-facing content without the ADR 0018 opt-in, and we accept that deliberately by bounding its lifetime instead of its content: a Pause Snapshot exists only while the Session is paused and is deleted on resume or terminal outcome, so no durable record of the model exchange accumulates and ADR 0018 remains the rule for anything long-lived. Alternatives rejected: keeping the paused process alive (work evaporates on crash or reboot, and liveness becomes the state model), forcing Debug Transcripts on for background Sessions (an append-only diagnostic log misused as state, with a strictly larger privacy surface because it never gets deleted), and rebuilding context from the Trace on resume (a lossy resume that silently breaks ADR 0027's one-task-one-Session promise).
