# A Queue Decision May Widen a Paused Session's Effect Envelope

ADR 0026 has the user state the Effect Envelope when the Session starts, and a strict reading would freeze it there. But one misjudged write scope then turns a background Session into a serial-pause loop — pause on the first out-of-scope file, decide, resume, pause minutes later on its sibling — which destroys the unattended value the envelope exists to enable. So the Decision Queue's vocabulary includes approve-and-widen: alongside approving or denying the single pending action, the user may admit the pending target's path prefix or command into the envelope for the remainder of the Session. Widening is only ever an explicit user decision recorded as Trace evidence like the original declaration, which keeps ADR 0026's real invariant — the boundary of durable effects is user-declared and evidenced — while relaxing the incidental one that it is declared exactly once. The envelope only widens mid-Session, never narrows or gets re-declared wholesale, and destructive or secret-touching actions stay denied regardless. Alternatives rejected: approve-once only (serial pauses, or users pre-declaring maximally wide envelopes, which is worse for the boundary's honesty), implicit widening on approval (the boundary grows without the user saying so — concealment-adjacent), and stopping to restart with a wider envelope (splits one task's evidence across Sessions, contra ADR 0027).
