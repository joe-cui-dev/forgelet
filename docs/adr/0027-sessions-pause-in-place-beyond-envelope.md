# A Session Pauses In Place When It Exceeds Its Effect Envelope

When a background Session needs an action outside its Effect Envelope, it pauses as the same Session and waits for a user decision, rather than completing and spawning a Session Continuation per ADR 0014. ADR 0014 governs continuing from completed Sessions; a pause is a mid-flight state, and appending pause, decision, and resume events to an append-only Trace does not mutate prior evidence. Keeping one task as one Session preserves the Effect Envelope, budget, and goal across the pause and keeps audit as one task, one evidence file. The alternative — one task shattering into a chain of continuations, each needing its envelope re-declared or a new inheritance rule — was rejected as worse on both usability and audit clarity.
